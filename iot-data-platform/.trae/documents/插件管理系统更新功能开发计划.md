# 插件管理系统更新功能开发计划

## 项目概述
在现有设备集成页面基础上，开发完整的插件管理和集成管理功能，包括插件更新、集成实例 CRUD、映射配置和调度配置等核心功能。

## 技术栈
- **前端框架**: Vue 3 + TypeScript + Composition API
- **UI 组件库**: Element Plus
- **状态管理**: Vue Reactivity API
- **构建工具**: Vite

---

## 一、插件管理更新功能

### 1.1 插件包替换功能
**目标**: 支持上传新插件包并覆盖现有版本

**实现步骤**:
1. 在插件列表操作列添加"更新"按钮（已有 `updatePlugin` 函数）
2. 开发插件更新对话框组件
   - 显示当前插件信息（名称、版本号、文件大小）
   - 提供新 JAR 文件上传区域
   - 支持版本号自动识别或手动输入
   - 添加更新日志输入框
3. 实现文件验证逻辑
   - 只支持 `.jar` 格式
   - 文件大小限制 50MB
   - 验证插件包完整性（可选：MD5 校验）
4. 调用后端更新接口（预留 API 调用位置）
5. 更新成功后刷新插件列表
6. 添加错误处理和用户反馈

**UI 组件**:
- `el-dialog` - 更新对话框
- `el-upload` - 文件上传组件
- `el-form` - 表单验证
- `el-progress` - 上传进度显示

**API 接口** (待对接):
- `PUT /api/system/iot/plugin/{id}/update` - 更新插件包

### 1.2 插件信息编辑功能
**目标**: 允许修改插件描述、版本号及其他元数据

**实现步骤**:
1. 在插件列表操作列添加"编辑"按钮
2. 开发插件编辑对话框
   - 插件名称（只读）
   - 版本号（可编辑）
   - 描述信息（可编辑）
   - 其他元数据字段
3. 表单验证
   - 版本号格式验证（语义化版本：x.y.z）
   - 必填项检查
4. 调用后端更新接口
5. 更新成功后刷新列表
6. 添加取消和确认机制

**数据结构**:
```typescript
interface PluginEditForm {
  id: string
  name: string // 只读
  version: string
  description: string
  metadata?: Record<string, any>
}
```

**API 接口** (待对接):
- `PUT /api/system/iot/plugin/{id}` - 更新插件信息

### 1.3 数据完整性和错误处理
**实现要点**:
1. 更新前备份原插件信息
2. 事务性操作：失败时回滚
3. 明确的错误提示
   - 网络错误
   - 文件格式错误
   - 版本冲突错误
   - 权限不足错误
4. 成功/失败消息反馈
5. 操作日志记录

## 二、集成管理功能开发

### 2.1 集成实例 CRUD 功能
**目标**: 实现完整的集成实例创建、读取、更新、删除

**实现步骤**:

#### 2.1.1 创建集成实例
1. 在集成列表添加"新增"按钮（已有 `addIntegration` 函数）
2. 开发集成实例创建对话框
   - 基本信息：名称、类型、描述
   - 选择插件：下拉选择已安装的插件
   - 连接配置：端点地址、认证信息
3. 表单验证
4. 调用后端创建接口
5. 创建成功后刷新列表

#### 2.1.2 查看集成实例
1. 在集成列表添加"查看"按钮
2. 开发集成实例详情对话框
   - 显示所有基本信息
   - 显示配置信息（敏感信息脱敏）
   - 显示映射配置
   - 显示调度配置
   - 显示运行状态

#### 2.1.3 更新集成实例
1. 在集成列表添加"编辑"按钮（已有 `editIntegration` 函数）
2. 复用创建对话框，填充现有数据
3. 支持修改所有可配置项
4. 调用后端更新接口

#### 2.1.4 删除集成实例
1. 在集成列表添加"删除"按钮（已有 `deleteIntegration` 函数）
2. 删除前确认对话框
3. 检查是否有依赖关系
4. 调用后端删除接口
5. 删除成功后刷新列表

**数据结构**:
```typescript
interface Integration {

  type: string

  status: 'connected' | 'disconnected' | 'error'
  config: Record<string, any>
  deviceMapping?: DeviceMapping
  timeSeriesMapping?: TimeSeriesMapping

```

**API 接口** (待对接):

- `DELETE /api/system/iot/integration/{id}` - 删除集成
- `GET /api/system/iot/integration/list` - 获取集成列表

#### 2.2.1 设备信息映射标签页
**功能**: 配置第三方系统设备与平台设备的映射关系

**实现步骤**:
1. 开发映射配置界面
   - 左侧：第三方系统设备字段列表（从插件读取）
   - 右侧：平台标准设备字段列表
   - 中间：映射关系连线或选择器
2. 实现字段映射逻辑
   - 字段名称映射
   - 数据类型转换
   - 默认值设置
   - 转换规则配置
3. 数据预览功能
   - 显示示例数据
   - 验证映射规则
4. 保存映射配置到 JSON 结构

**数据结构**:
```typescript
interface DeviceMapping {
  mappings: FieldMapping[]
  transformRules?: TransformRule[]
}

interface FieldMapping {
  sourceField: string
  targetField: string
  dataType: 'string' | 'number' | 'boolean' | 'date'
  defaultValue?: any
  transformFunction?: string
  required: boolean
}

interface TransformRule {
  field: string
  ruleType: 'trim' | 'uppercase' | 'lowercase' | 'regex' | 'custom'
  ruleValue?: string
}
```

#### 2.2.2 时序数据映射标签页
**功能**: 配置时序测点数据的映射关系

**实现步骤**:

3. 支持表达式配置
   - 简单数学运算
   - 条件判断
   - 函数调用
4. 保存映射配置

**数据结构**:
```typescript
interface TimeSeriesMapping {
  pointMappings: PointMapping[]
  aggregationRules?: AggregationRule[]
}

interface PointMapping {
  sourcePointCode: string
  targetPointCode: string
  dataType: 'analog' | 'digital' | 'accumulated'
  unitConversion?: UnitConversion
  precision?: number
  expression?: string
}

interface UnitConversion {
  fromUnit: string
  toUnit: string
  formula: string
}

interface AggregationRule {
  pointCode: string
  aggregationType: 'avg' | 'max' | 'min' | 'sum'
  interval: number
  intervalUnit: 'second' | 'minute' | 'hour'
}
```

interface StatusMapping {
  statusMappings: StatusMappingItem[]

interface StatusMappingItem {

## 六、技术难点和解决方案

### 6.1 映射配置界面复杂度
**难点**: 多对多映射关系复杂，用户操作繁琐

**解决方案**:
- 使用拖拽式映射界面
- 提供批量映射功能
- 支持搜索和过滤
- 提供映射模板

### 6.2 数据转换性能
**难点**: 大量数据转换可能影响性能

**解决方案**:
- 使用 Web Worker 进行后台转换
- 实现增量转换
- 添加转换缓存
- 优化转换算法

### 6.3 调度任务管理
**难点**: 多个集成实例的调度任务冲突

**解决方案**:
- 使用任务队列管理
- 实现任务优先级
- 添加任务去重机制
- 使用防抖和节流
